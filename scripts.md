# AI 賦能全端開發 — Hahow 線上課宣傳影片腳本

> **影片長度**：2 分鐘（120 秒）
> **講師**：凱文大叔 Uncle Kevin
> **語調**：務實、自信、不堆砌名詞，像資深同事跟你分享實戰經驗
> **目標觀眾**：有 Java 基礎的後端 / 全端工程師，想把 AI 放進真實系統

---

## 第一段：痛點共鳴（0:00 – 0:25）

**【畫面】** 工程師螢幕快速切換：ChatGPT 對話視窗、散落的 demo 專案、Stack Overflow 搜尋頁面。畫面逐漸變灰、靜止。

**【口白】**

> 你花了一個週末串好 LLM API，聊天機器人跑起來了——然後呢？
>
> 沒有資料庫，沒有登入，沒有權限控管。
> AI 回答的金額是瞎猜的，推薦的產品根本不存在。
> 主管問你「這能上線嗎？」——你知道答案是不能。
>
> 問題不是你不會呼叫 API，而是你缺一條完整的路，
> 把 AI 放進一套真正能用的企業系統裡。

## 第二段：課程解法（0:25 – 0:55）

**【畫面】** IDE 畫面全螢幕，左下角浮現講師資訊卡片（頭像圖 + 名稱「凱文大叔 Uncle Kevin」），卡片下方依序淡入事跡標籤：「YouTube 14K+ 訂閱」→「著作《使用 Spring AI 打造企業 RAG 知識庫》」→「Hahow 課程《Dify 零代碼革命》」，每個標籤停留 2 秒。講師卡片淡出後，程式碼片段快速滾動：Spring Boot Controller → JPA Entity → React Dashboard → AI 聊天室串流回應，底部浮現技術標籤（Spring Boot 4 / Spring AI 2.0 / React 19 / pgvector）。

**【口白】**

> 我是凱文大叔，《使用 Spring AI 打造企業 RAG 知識庫》的作者，
> YouTube 頻道超過一萬四千訂閱。
> 你可能也看過我在 Hahow 的另一門課——Dify 零代碼革命。
>
> 這一次，我要帶你用一套 AI CRM 實戰專案，
> 透過 AI 從第一行程式碼到可執行系統完整跑通。
>
> 八個單元，章章相連——
> 先用 Spring Boot 4 建好後端骨架，
> 接上 PostgreSQL 儲存資料、Flyway 管理資料庫遷移，
> 再用 Spring Security 加上 JWT 登入與角色權限。
>
> 前端用 React 建出 CRM Dashboard、客戶列表和商機看板，
> 然後——重頭戲來了——
> 用 Spring AI 2.0 接入大語言模型，
> SSE 串流回應，再用 Tool calling 讓 AI 去查資料庫裡的真實客戶資料。
>
> AI 回答的每一個金額、每一筆商機，都是從系統裡查出來，不是瞎猜的。

## 第三段：差異化賣點（0:55 – 1:20）

**【畫面】** 分割畫面對比：左邊是一般課程（只有一個聊天視窗），右邊是本課程的 CRM 系統全貌（Dashboard → 客戶詳情 → AI 摘要卡片 → RAG 引用來源 → Swagger 文件）。右邊畫面逐步展開，左邊逐漸縮小淡出。

**【口白】**

> 這門課跟你之前看到的 AI 課不一樣。
>
> 第一，所有金額、次數、權限——由 Java 的 domain service 算；
> AI 只負責摘要、建議和文字草稿。數字由工具算，內容由模型寫。
>
> 第二，RAG 不是玩具。我們用 pgvector 建真正的知識庫，AI 推薦產品的時候，會附上引用來源，查無資料就明說。
>
> 第三，每一章都有 AI Agent 提示詞範本，你可以直接貼給 Claude Code 或 Codex，讓 AI 幫你產程式、你負責驗證——這就是現代工程師的開發方式。
