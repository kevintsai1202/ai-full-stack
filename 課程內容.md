# AI 賦能全端開發：從零打造企業級智慧應用

> 課程英文名：AI-Empowered Full-Stack: Building Enterprise-Grade Smart Apps

## 1. 課程定位

本課程整合 `D:\GitHub\learn-spring` 與 `D:\GitHub\learn-embabel` 的內容，重新設計為一門以「AI CRM」為主軸的全端實戰課。學員會從零建立一套企業級 CRM 系統，先完成 Spring Boot 後端、React 前端、資料庫、認證授權與 API 文件，再逐步加入 Spring AI、RAG、工具呼叫、MCP 與結訓專案整合，最後完成可登入、可查詢客戶、可產生 AI 客戶摘要、可產生銷售建議的完整系統。

這不是把多門課串在一起，而是用同一套 AI CRM 範例把每個技術放進同一條產品開發路線中。每一章的產出都會成為後續章節的基礎，避免學員只學到零散範例。

## 3. 課程對象

- 已具備 Java 基礎，想把 Spring Boot、React 與 AI 串成完整產品的工程師。
- 會寫基本 REST API，但還不熟悉企業級分層、權限、錯誤處理、資料庫遷移與前後端整合的開發者。
- 想理解 Spring AI 與 RAG 如何落地在真實 JVM 系統，而不是只做聊天機器人 demo 的後端工程師。
- 想用 AI Agent 協作開發，但希望產出可測試、可維運、可上線程式碼的全端工程師。

## 4. 學習成果

完成課程後，學員能夠：

- 從零建立 Spring Boot + React 的 AI CRM 專案骨架。
- 用 Spring MVC 設計客戶、商機、互動紀錄與任務管理 API。
- 用 Spring Data JPA、Specification 與 Flyway 建立可維護的資料模型。
- 用 Spring Security + JWT 完成登入、角色與 API 授權。
- 用 React 建立 CRM Dashboard、客戶列表、商機看板與 AI 助理介面。
- 用 Spring AI ChatClient 建立可串流回應的 CRM 助理。
- 用 pgvector 建立客戶互動紀錄、產品文件與銷售話術的 RAG 檢索。
- 用 tool calling 讓 AI 讀取 CRM 的真實 domain data，而不是憑空回答。
- 用 MCP 擴充外部工具，例如行事曆、郵件草稿、報表匯出或第三方 CRM。
- 完成結訓專案整合，從登入到 AI 對話完整跑通並在 Demo Day 驗收。
- 用測試、log、audit trail 與上線檢查清單驗證 AI 功能的可靠性。

## 5. 貫穿案例：AI CRM 智慧業務助理

本課程的範例系統是一套 B2B AI CRM，目標使用者是業務團隊、客服主管與銷售經理。系統從基本 CRM 功能開始，逐步加入 AI 能力。

### 5.1 業務情境

公司已有客戶、聯絡人、商機、訂單、客服紀錄與產品知識庫。業務希望打開客戶頁面時，系統可以自動整理客戶狀態：

- 這個客戶最近是否活躍？
- 最近互動紀錄中有哪些風險訊號？
- 是否有即將流失或續約延遲的可能？
- 哪些產品或方案最適合推薦？
- 下一步應該寄信、安排會議、請主管介入，還是先補資料？

課程中的 AI 不會直接替代人員決策，而是提供可追蹤的建議。所有金額、訂單、互動次數與權限判斷都由 Java domain service 或資料庫提供；LLM 負責摘要、分類、建議與文字草稿。

### 5.2 系統模組

| 模組 | 主要功能 | 對應技術 |
|---|---|---|
| 認證與權限 | 登入、JWT、角色權限、API 保護 | Spring Security |
| 客戶管理 | 客戶、聯絡人、標籤、分級 | Spring MVC + JPA |
| 商機管理 | 商機階段、金額、預計成交日、風險狀態 | JPA + Specification |
| 互動紀錄 | 電話、會議、Email、客服紀錄 | JPA + Flyway |
| AI 摘要 | 客戶摘要、風險訊號、下一步建議 | Spring AI |
| RAG 知識庫 | 產品文件、銷售話術、服務條款檢索 | pgvector |
| 工具呼叫 | 讀取真實 CRM 資料、計算商機健康分數 | Spring AI Tool Calling |
| 結訓整合 | 端到端測試與 Demo Day 驗收 | — |
| 前端工作台 | Dashboard、列表、看板、AI Chat | React |

## 7. 詳細課綱

### Unit 1：開發環境、專案骨架與 AI 協作流程

**學習目標**

- 建立 Windows + PowerShell 7+ 的 Java / Node / Docker 開發環境。
- 建立 `backend` 與 `frontend` 分離的 full-stack 專案。
- 理解課程中 AI Agent 協作的使用方式與限制。
- 完成第一個 Spring Boot 啟動點與 React 啟動頁。

**課程內容**

- JDK 21、Maven、Node.js、Git、Docker Desktop 安裝與版本驗證。
- Spring Initializr 建立後端專案。
- Vite + React + TypeScript 建立前端專案。
- 後端、前端、資料庫的目錄結構規劃。
- `.env`、`application.yml`、PowerShell 啟動指令整理。
- AI Agent 協作規範：先讀需求、再產程式、最後驗證。

**實作任務**

- 建立 `ai-crm/backend` 與 `ai-crm/frontend`。
- 後端新增 `/api/health`。
- 前端顯示 API health 狀態，連線失敗時顯示紅燈。
- 撰寫第一份 AI Agent 提示詞：請 AI 幫你檢查本機環境。

**AI Agent 提示詞主題**

- 檢查 Windows 開發環境。
- 產生 Spring Boot + React 專案初始化指令。
- 產生 PowerShell 啟動與驗證腳本。

**驗收標準**

- `mvn spring-boot:run` 可啟動後端。
- `npm run dev` 可啟動前端。
- 前端可正確顯示後端健康狀態。

### Unit 2：Spring MVC、REST API 與 CRM Domain Modeling

**學習目標**

- 理解 Spring MVC 請求流程與 Controller / Service / Repository 分層。
- 設計 CRM 核心 domain model。
- 建立客戶、聯絡人、互動紀錄的 REST API。
- 用 Bean Validation 保護輸入資料。

**課程內容**

- DispatcherServlet 與 Spring MVC 請求生命週期。
- REST API 設計：URL、HTTP method、status code、DTO。
- CRM domain 拆解：Customer、Contact、Interaction、Opportunity。
- Controller / Service 分工原則。
- Request DTO、Response DTO 與 Entity 的邊界。
- Bean Validation 與錯誤訊息設計。

**實作任務**

- 建立 `CustomerController`、`CustomerService`。
- 建立新增客戶、查詢客戶、更新客戶狀態 API。
- 建立互動紀錄 API：新增電話、會議或 Email 紀錄。
- 用 PowerShell `Invoke-RestMethod` 驗證 API。

**AI Agent 提示詞主題**

- 從 CRM 需求產生 REST API 草案。
- 產生 DTO、Controller 與 Service 的初版程式。
- 檢查 API 命名是否符合 REST 原則。

**驗收標準**

- 可新增客戶並查詢回來。
- 驗證失敗時回傳可讀的 400 錯誤。
- Controller 不直接處理商業邏輯。

### Unit 3：PostgreSQL、Flyway、JPA 與動態查詢

**學習目標**

- 用 Docker Compose 啟動 PostgreSQL 與 pgvector。
- 用 Flyway 管理 schema migration。
- 用 Spring Data JPA 建立 Entity 與 Repository。
- 用 Specification 建立 CRM 搜尋條件。

**課程內容**

- 為什麼企業專案不能只靠記憶體資料。
- Docker Compose 啟動 PostgreSQL。
- Flyway 命名規則與 migration 策略。
- Entity mapping、關聯設計與 audit 欄位。
- Repository query method 與 Specification 的選用。
- 交易邊界：`@Transactional` 放在 Service。

**實作任務**

- 建立 `customers`、`contacts`、`interactions`、`opportunities` 資料表。
- 將 Unit 2 的記憶體服務改成 JPA。
- 建立客戶搜尋 API：依產業、分級、負責業務、最近互動時間查詢。
- 新增 audit 欄位：createdAt、updatedAt。

**AI Agent 提示詞主題**

- 根據 Entity 產生 Flyway migration。
- 根據查詢需求產生 Specification。
- 檢查 JPA 關聯是否可能造成 N+1 查詢。

**驗收標準**

- 重啟應用後資料仍存在。
- Flyway 可正常執行 migration。
- 搜尋 API 可組合多個條件。

### Unit 4：Spring Security、JWT、OpenAPI 與企業級錯誤處理

**學習目標**

- 建立登入、JWT 簽發與認證過濾器。
- 設計角色權限：業務、主管、管理員。
- 建立全域例外處理與一致錯誤格式。
- 產生可供前端與測試使用的 OpenAPI 文件。

**課程內容**

- Spring Security filter chain 基礎。
- Password hashing 與測試帳號 seed。
- JWT payload、角色與過期時間。
- `SecurityConfig`、permit list 與 protected API。
- `@ControllerAdvice` 與 `ProblemDetail`。
- Swagger UI 與正式環境開關。

**實作任務**

- 建立 `/api/auth/login`。
- 建立 JWT filter。
- 保護 `/api/customers/**` 與 `/api/opportunities/**`。
- 在 Swagger 顯示 Bearer auth。
- 建立統一錯誤格式。

**AI Agent 提示詞主題**

- 產生最小可用 JWT 認證流程。
- 檢查哪些 API 應該公開、哪些需要登入。
- 產生 OpenAPI 標註與錯誤回應範例。

**驗收標準**

- 未登入存取受保護 API 會回傳 401。
- 不同角色看到的 API 權限不同。
- Swagger 可帶 JWT 呼叫 API。

### Unit 6：Spring AI ChatClient、SSE 與 tool calling

**學習目標**

- 用 Spring AI ChatClient 建立 AI CRM 助理。
- 用 SSE 將 AI 回應串流到 React。
- 讓 AI 透過 tool calling 讀取真實 CRM 資料。
- 避免 LLM 憑空計算金額、次數與商機狀態。

**課程內容**

- ChatClient 基礎與 system prompt 設計。
- Prompt template 與上下文邊界。
- SSE endpoint 設計。
- React 串流訊息顯示。
- Tool calling：讓 Java service 提供客戶統計、最近互動、商機健康分數。
- AI 回答的信任邊界：數字由工具算，文字由模型寫。

**實作任務**

- 建立 `/api/ai/chat` 串流 endpoint。
- 前端建立 AI 助理聊天室。
- 建立 `CrmInsightTools`。
- AI 可回答「幫我整理客戶 A 的近況與下一步建議」。

**AI Agent 提示詞主題**

- 產生 ChatClient 設定與 system prompt。
- 產生 SSE 後端與 React 前端串流處理。
- 根據 CRM service 設計最小工具清單。

**驗收標準**

- 前端可看到 token-by-token 串流回應。
- AI 回答能引用真實客戶資料。
- 若工具查無客戶，AI 必須清楚說明無資料，不可自行編造。

## 10. 教學節奏建議

每個單元都採用同一個節奏：

1. 原理講解：先說明技術解決什麼問題。
2. 系統情境：把技術放回 AI CRM 需求。
3. 程式範例：由講師帶著建立最小可用版本。
4. AI Agent 提示詞：讓學員用 AI 協助產生或檢查程式。
5. 驗證情境：用 PowerShell、Swagger、前端畫面或測試確認結果。
6. 常見錯誤排查：補上 Windows、Java、Maven、Docker、JWT、SSE、RAG 與整合常見問題。

## 11. 與來源課程的整合方式

| 來源 | 原始重點 | 新課程中的位置 |
|---|---|---|
| `learn-spring` Day 1 | Spring Boot、MVC、JPA、Flyway、Security、API 文件、錯誤處理 | Unit 1-4 |
| `learn-spring` Day 2 | Spring AI、tool calling、RAG、MCP、React、SSE、完整系統驗證 | Unit 5-7 |

## 12. 課程賣點文案

這門課不是只教你「怎麼呼叫 LLM API」，而是帶你從零打造一套真正能放進企業系統的 AI 全端應用。你會學到 Spring Boot、Spring MVC、Spring Data JPA、Spring Security、Spring AI 與 React 如何各司其職，並用一個完整 AI CRM 專案串起資料庫、權限、前端、RAG、工具呼叫與端到端整合。

課程結束時，你不只會有一個聊天機器人，而是一套可登入、可查資料、可引用知識庫、可產生業務建議、可追蹤 AI 執行過程的企業級智慧應用。
